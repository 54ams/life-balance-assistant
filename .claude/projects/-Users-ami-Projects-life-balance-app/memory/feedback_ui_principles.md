---
name: UI design principles
description: User wants professional, story-driven UI with layman's language — no technical jargon, flow-based insights
type: feedback
---

- All insights/analytics must use layman's English — no correlation coefficients, no "r=0.42", no "FDR correction" visible to users. Translate into plain statements like "Your sleep and mood seem to move together."
- Date format: dd/mm/yyyy (London-based user)
- Don't show Profile tab in navigation — profile button is on the home screen instead
- Insights should flow like a story (Spotify Wrapped style) — not text-heavy cards with stats
- The app is for everyday people, not technically proficient users
- UI should feel like "big league" established apps — seamless flow, professional polish
- Keep things peaceful and easy to digest

**Why:** This is a dissertation prototype that will be used by 5-8 real participants and evaluated by examiners. The UX must demonstrate professional-grade design thinking.

**How to apply:** Every screen should be reviewed from a non-technical user's perspective. Replace jargon with plain English. Use visual storytelling over data tables. Prioritise flow and clarity.
